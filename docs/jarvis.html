Project Jarvis: Vitals & AI Correlation Engine

Goal: Transform Devansh OS from a reactive dashboard into a proactive, intelligent health and productivity assistant. The system will ingest temporal vitals data and correlate it with existing productivity metrics (coding, gym, deep work, sleep) to generate actionable insights.

Phase 1: Ingestion (Vitals API Integration)

Connect to a hardware health tracker (Fitbit, Oura, or Apple HealthKit) to pull high-fidelity physiological data.

  • Target Metrics: Resting Heart Rate (RHR), Heart Rate Variability (HRV), Sleep Stages (Deep/REM), Body Temperature, and Activity Levels.
  • Architecture: Create a new provider in app/providers/vitals.py to handle OAuth and daily synchronization of time-series data.

Phase 2: Data Normalization

Ensure all disparate data sources (commits, Leetcode, gym sessions, sleep logs, and new vitals) share a common chronological baseline.

  • Store vitals in the SQLite DB with strict ISO timestamps.
  • Create daily aggregation views to easily query a specific day's overall "Health Score" vs "Productivity Score".

Phase 3: The Correlation Engine (AI)

Pass the normalized week-over-week data to an LLM (Claude/GPT-4) with a system prompt designed to find hidden correlations.

  • Example input to LLM: "User logged 5 days of low deep work. HRV is down 15%. Sleep duration is normal but REM is low. Gym intensity was high early in the week."
  • Expected output: "You are showing signs of central nervous system fatigue. Your intense gym sessions early in the week compromised your sleep quality, leading to poor focus. Take 2 rest days."

Phase 4: Proactive Alerts (UI Integration)

Bring the insights into the UI so they aren't hidden behind menus.

  • Add a "Jarvis Insight" banner at the top of the Devansh OS web interface that displays the daily AI-generated observation.
  • Integrate with the Neglect panel to warn about impending burnout or illness before it happens.

Status: Planned for future iterations once hardware tracker is acquired.
